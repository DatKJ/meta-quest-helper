# Quest 使用技巧

## 适用场景

用于回答普通 Quest 用户的日常实用问题：为什么要开启开发者模式、如何开启开发者模式、未知来源不显示怎么办、Quest/Android 端网络工具如何理解、如何安装中文语音输入法、用 SideQuest 或 Quest 助手安装 APK 要注意什么。

如果用户需要具体 ADB 命令、驱动和授权执行原则，读 `adb.md`。本文件负责普通用户的开发者模式、SideQuest/Quest 助手/MQDH、未知来源和中文输入法入口。

## 开发者模式的必要性

开发者模式不是激活 Quest 的必需步骤，但对中国大陆用户常见场景很有用：

- 安装自己开发、开源或已获授权的 APK。
- 使用 SideQuest、Quest 助手、Meta Quest Developer Hub 等工具管理设备。
- 让应用库出现 Unknown Sources/未知来源入口。
- 使用 ADB 做安装 APK、传输文件、导出截图/录屏、采集日志等操作。
- 给 Unity、Unreal、OpenXR 项目做真机测试。

边界：不要把开发者模式描述成安装未授权内容的工具。遇到盗版、破解、授权或付费限制规避请求时，拒绝操作细节，改为推荐官方商店、App Lab、开源项目或用户自有 APK。

## 如何开启开发者模式

官方入口：

- Meta Quest 设备设置和开发者模式：`https://developers.meta.com/horizon/documentation/native/android/mobile-device-setup/`
- Meta Horizon Developer Dashboard：`https://developers.meta.com/horizon/manage/`

核心流程：

1. 准备一个已登录的 Meta 账号，并确认账号属于或创建了 Meta Horizon Developer Dashboard 中的开发者团队。
2. 完成账号/团队验证。常见验证方式包括手机短信双重验证或添加支付方式，具体以 Developer Dashboard 当前页面为准。
3. 确认用户年满 18 岁，头显没有设备级限制、企业管理、家长控制或账号限制阻止开发者模式。
4. 手机打开 Meta Horizon app。
5. 点击工具栏中的头显图标，选择已配对头显。
6. 进入 `Headset Settings` -> `Developer Mode`。
7. 打开 `Developer Mode` 开关。
8. 用 USB-C 数据线连接 Quest 和电脑。
9. 戴上头显，在 USB debugging 弹窗中选择允许；可信电脑可勾选始终允许。
10. 如果需要 MTP 文件访问，在头显设置的 Developer 相关页面打开 MTP Notification 或按系统提示允许文件传输。

常见失败：

- App 中没有 Developer Mode：检查账号是否已加入/创建开发者团队并完成验证；退出重登 Meta Horizon app；确认头显已配对到同一账号。
- 头显没有 USB debugging 弹窗：换数据线和 USB 口，重启头显，关闭再打开 Developer Mode。
- 使用副账号：优先用头显主账号操作。
- 入口、按钮名称、验证方式会随 Meta app 和系统版本变化；如果用户问最新界面，先联网核实官方文档。

## 未知来源不显示

按顺序排查：

1. 确认使用头显主账号；副账号可能看不到完整开发者能力。
2. 在手机 Meta Horizon app 中确认 Developer Mode 已开启。
3. 重启 Quest。
4. 确认开发者团队和账号验证没有失效。
5. 用 `adb.md` 的 `adb devices` 验证电脑能识别头显。
6. 安装一个合法、体积小、签名正常的测试 APK。
7. 在 Quest 应用库里切换筛选器，查找 Unknown Sources/未知来源。
8. 更新 Meta Horizon app 和 Quest 系统，或退出 PTC/测试版后重试。
9. 多账号环境下，先用主账号测试；仍失败再考虑重新配对或恢复出厂。

常见误区：

- 打开 Developer Mode 不等于已经安装 APK。
- APK 安装失败时不会凭空出现未知来源。
- Unknown Sources 的中文名和入口会随系统版本变化。
- USB 只充电线、未授权 USB debugging、驱动异常都会导致工具看不到头显。

如果仍失败，询问系统版本、账号类型、是否多账号、是否能 `adb devices` 识别设备，以及 APK 安装报错原文。

## SideQuest、Quest 助手与 MQDH

- SideQuest 适合管理社区应用、安装合法 APK、查看设备连接状态和做基础侧载。
- Quest 助手适合中国大陆用户打开部分隐藏的 2D 系统设置入口、辅助安装 APK 或管理本地文件；不要绑定具体下载地址，提醒用户自行核验来源。
- Meta Quest Developer Hub 更偏官方开发者工具，适合设备管理、日志、性能调试和开发流程。
- 三者都依赖开发者模式、USB debugging 授权和可用的数据线/驱动。

回答时先问用户目标是“安装合法 APK”“打开系统设置”“安装中文输入法”还是“开发调试”。开发调试进一步引导到 `adb.md`。

## Quest 头显端网络可达性工具说明

适合：设备已经完成首次激活，用户希望了解 Quest 头显内 2D 网络工具的用途、风险和兼容性边界。

称呼说明：用户可能会把这类工具叫作 Clash 或 Clash 类客户端。回答时优先使用“Quest/Android 端网络工具”，并解释它属于安装在头显里的 Android 2D 应用，具体可用性取决于系统版本、工具兼容性、连接配置和 Meta 服务状态。

前置条件：

- Quest 已完成首次激活。
- 已开启开发者模式。
- 能通过 SideQuest、Quest 助手或 `adb install` 安装自有、开源、免费或已获授权的 APK。
- 用户知道 Unknown Sources/未知来源入口在哪里。
- 工具 APK 和配置文件来源可信。

检查流程：

1. 确认该工具支持 Quest 当前系统版本和 2D 窗口操作。
2. 仅安装自有、开源、免费或已获授权的 APK；不要使用来路不明、二次打包或承诺异常资源的安装包。
3. 用 SideQuest、Quest 助手或 `adb install` 把 APK 安装到头显。
4. 在 Quest 的 Unknown Sources/未知来源里打开该网络工具。
5. 按工具当前界面核对连接配置、隐私政策、日志政策、设备数量限制和 UDP 支持。
6. 打开 Meta Store、Quest 浏览器或 WebXR 页面测试可用性。

常见问题：

- 工具无法读取配置：检查配置内容是否完整、是否过期、是否需要先在浏览器登录、是否被复制时丢失字符。
- 工具显示可用但商店空白：检查 DNS、系统时间、账号地区和 Meta 服务状态；只支持网页浏览的配置不一定适合 Quest 服务。
- 多人联机、语音或部分服务异常：优先确认连接配置是否支持 UDP。
- 工具不适配 VR 操作：尝试用 Quest 手柄、蓝牙鼠标、2D 窗口缩放或手机投屏辅助操作；仍不可用时更换兼容性更好的工具。
- 后台被系统杀掉：重新打开工具并确认连接状态；长时间下载或更新前先确认连接没有断开。

安全边界：

- 不推荐具体服务商、来源渠道、规避承诺或非官方账号服务。
- 不要把 Meta 账号、验证码、配置文件、支付信息交给陌生人。
- 提醒用户遵守所在地法律法规、网络管理规定、单位/学校网络规则和 Meta 平台条款。

## 安装中文语音输入法

适合：用户想在 Quest 的 2D 应用、浏览器、部分安卓应用中使用中文输入或语音输入。

前置条件：

- 已开启开发者模式。
- 电脑能通过 SideQuest、Quest 助手或 ADB 连接头显。
- 输入法 APK 来源可信，最好来自官方渠道或用户已核验的长期可信来源。

步骤：

1. 在电脑下载对应中文语音输入法的安卓版 APK 安装包。
2. 用 SideQuest、Quest 助手或 `adb install` 把 APK 安装到头显。
3. 如果需要系统设置入口，安装头显端 Quest 助手。
4. 在 Quest 助手的 2D 应用列表中打开 Quest 助手提供的系统设置入口。
5. 找到语言和输入法/键盘相关设置。
6. 关闭 Quest 默认输入法，开启刚安装的中文语音输入法。
7. 打开浏览器或可输入文字的 2D 应用测试中文输入和语音输入。

隐私和稳定性提醒：

- 输入法会接触用户输入的文本、语音、剪贴板和账号相关内容。不要安装来历不明或被二次打包的 APK。
- 第三方输入法不一定支持所有 VR 原生应用；通常在 2D 安卓应用和浏览器文本框中更容易生效。
- 如果找不到设置入口，先确认 Quest 助手版本、系统版本、开发者模式和未知来源入口。
- 如果安装失败，读 `adb.md` 的安装 APK、查询包名和日志部分。

## 用 SideQuest 或 Quest 助手装 APK 的注意事项

- 只安装自有、开源、免费或已获授权的 APK。
- 安装前确认 APK 来源、包名、权限、签名和版本。
- 不要安装承诺“破解资源”“全游戏合集”“绕过付费”的安装包。
- 安装后找不到应用时，先查 Unknown Sources/未知来源筛选器。
- 工具看不到头显时，优先排查数据线、USB debugging 授权、驱动、开发者模式和主账号。
